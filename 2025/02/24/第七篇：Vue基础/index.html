Vue基础

组件通信

Vue组件之间有哪些通信方式？

组件通信的基本原则：谁向谁传，数据写在发送方。主要通信方式包括：

  1. 父传子：props
  2. 子传父：$emit
  3. 兄弟组件：
    • $attrs/$emit
    • $parent/$root
    • Vuex
    • Event Bus($emit/$on)

注：$attrs 是一个包含父组件传递给子组件但未在子组件的 props 中定义的属性的对象。

指令优先级

v-if和v-for的优先级是怎样的？

  • Vue2中v-for优先级更高
  • Vue3中v-if优先级更高
  • 官方文档明确表明不要在同一个标签中同时使用这两个指令

生命周期

Vue的生命周期有哪些？各自的作用是什么？

Vue2生命周期：

  1. beforeCreate

    • 在实例初始化之后，数据观测和事件配置之前调用
    • 用途：初始化操作，但不能访问响应式数据

  2. created

    • 实例创建完成后调用
    • 已完成数据观测、属性和方法的运算，watch/event事件回调
    • 用途：数据获取、事件监听等初始化操作

  3. beforeMount

    • 挂载开始前调用
    • 用途：渲染前的准备工作，如数据格式化

  4. mounted

    • DOM挂载完成后调用
    • 用途：DOM操作、第三方库初始化

  5. beforeUpdate

    • 数据更新时调用，发生在虚拟DOM打补丁之前
    • 用途：更新前访问现有DOM，如移除事件监听器

  6. updated

    • 虚拟DOM重新渲染和打补丁之后调用
    • 用途：更新后的DOM操作，但应避免在此阶段更改状态

  7. beforeDestroy

    • 实例销毁前调用
    • 用途：清理操作，如移除事件监听器、定时器

  8. destroyed

    • 实例销毁后调用
    • 用途：最终的清理工作

Vue3新增：

  • setup
    • 在组件实例创建之前执行
    • 组合式API的入口
    • 可访问props和context，但不能访问this

组件扩展

如何扩展一个组件？

  1. 使用Composition API进行逻辑扩展
  2. 使用slots进行内容扩展
    • 默认插槽
    • 具名插槽
    • 作用域插槽

虚拟DOM

什么是虚拟DOM？有什么优点？

虚拟DOM是一个虚拟的DOM对象，本质是一个JS对象。

优点：

  1. 减少操作真实DOM的次数，提高程序性能
  2. 方便实现跨平台

Vue3新特性

Vue3相比Vue2有哪些重要的新特性？

  1. Composition API

    • 用于组织和复用组件逻辑
    • 将相同功能的代码放在一起，便于管理

  2. 响应式系统升级

    • 使用Proxy替代Object.defineProperty
    • 更好的性能和更完整的响应式支持

  3. 动态路由

    • 更灵活的路由配置
    • 在params后添加反斜杠和冒号实现动态路由
